# 从论文到实现：逻辑论证

## 论文 1：Li et al. (2025) — Score ID Bias

### 论文发现
当 prompt 中写 `Score 1=Poor, Score 5=Excellent`，LLM 会锚定在数字标签上。用字母 {E,D,C,B,A} 替代 {1,2,3,4,5} 后，评分分布显著变化。

### agent-skill-infra 的对应问题
我们的旧 prompt：
```
0.3=mentioned but vague. 0.5=adequate. 0.7=clear and specific.
```
这给 LLM 提供了三个"安全默认值"。LLM 面对大多数"还可以"的 skill 时，选择中间值 0.5 → 维度平均 0.57 → 最终 57%。

**证据**：29 个 OpenClaw skill，10 个精准落在一模一样的 57%。这不是巧合。

### 具体修复
删除所有 `X.X=label` 格式的数字锚点。改为：
```
Use the full 0-1 range. Avoid identical scores across dimensions.
```

### 预期效果
57% 集群消失。评分分布更均匀。

### 实际验证
57% 出现次数：10/29 (34%) → 1/27 (4%)。✅ 论文预测成立。

## 论文 2：Xu et al. (2026) — Position Bias

### 论文发现
评分标准在 prompt 中的位置影响其权重。第一条标准被过度重视，最后一条被忽视。排列校准（多次运行，每次排列不同，取均值）可以消除该偏差。

### agent-skill-infra 的对应问题
我们的 prompt 中 8 个维度固定排列：
```
1. trigger_precision  ← 总是第一位，可能被过度加权
2. output_completeness
...
8. edge_cases         ← 总是最后一位，可能被忽视
```

这导致：trigger_precision 的评分可能系统性地偏高（不是因为 skill 写得好，而是因为它是第一条标准）。

### 具体修复
每次 LLM 调用时，随机排列 8 个维度的顺序。对同一 skill 跑 3 次（温度 0.1），取均值。均值消除了排列偏差的影响。

### 预期效果
1. 单个维度的评分不再受位置影响
2. 均值 ± 温度误差仍然是稳定区间
3. 不会降低准确性（Xu et al. 证明排列校准提升人工相关性）

### 为什么必须温度 0.1 而非 0.0
温度=0.0 下，LLM 对同一 prompt 返回完全相同的结果 → 排列变化无效果。温度=0.1 下，排列差异会产生 1-2% 的微小分数变化，取均值后消除。

## 论文 3：Lee et al. (2025) — Calibration Framework

### 论文发现
LLM 评分有系统性偏差（特异性 ≠ 灵敏度）。需用人类校准集修正。提供了一个可安装的 pip 包。

### agent-skill-infra 的对应问题
我们的 LLM Judge 评估的是 SKILL.md 质量，但没有任何校准。我们不知道：
- 灵敏度：LLM 判定一个"好 skill"为好 skill 的概率？
- 特异性：LLM 判定一个"差 skill"为差 skill 的概率？
- 偏差修正后的真实质量分数是什么？

### 为什么现在不做
校准需要人类标注的 ground truth（至少 10 个人类评审的 skill 分数）。我们目前没有这个数据集。这是 P2 而非 P0。

### 何时做
当有 10+ 个人类标注的 skill 质量分数后，作为可选的校准层集成。

## 论文 4：三态阈值的逻辑基础

### 论文支撑
Li et al. 和 Xu et al. 共同指向一个结论：单一 LLM 评分不可靠。必须用多次评分的均值 + 分布信息来决策。

### 我们的方法
基于 n=27 的 OpenClaw skill 分布：

```
分布: 41-82, μ=68.1, σ=8.4
P25=64, P50=70, P75=74
```

但阈值不能只基于 OpenClaw 数据，因为 OpenClaw skill 本身的质量分布不能代表"所有 skill"。需要混合 Gold + Random 两组数据后设定阈值。

### 当前能做的
1. 先实现维度排列（消除最后一个已知偏差源）
2. 用排列 + 温度 0.1 重新跑 30 skill 的完整数据
3. 基于完整数据设定阈值
4. 阈值标注为 "基于 n=30 样本的经验阈值，随数据增长调整"


**当前最合理的单一步骤**：实现维度排列 → 用排列方式重跑 30 skill → 基于无偏数据设定三态阈值。
